# 安全策略

## 支持范围

当前主分支（`main`）是安全修复优先支持版本。历史分支不保证持续接收补丁。

## 漏洞报告

请不要在公开 Issue 直接披露高风险漏洞细节。建议私下联系维护者并提供：

- 漏洞类型与影响范围
- 复现步骤或 PoC（最小化）
- 受影响版本/分支
- 修复建议（可选）

## 响应流程（建议目标）

1. 24-72 小时内确认收到
2. 评估风险并给出处理计划
3. 完成修复后发布公告/变更说明

## 安全开发建议

- 不将密钥写入代码仓库
- 使用 `.env` 管理配置，并确保 `.gitignore` 生效
- 对工具输入做参数校验
- 对文件写入/编辑保留冲突检测与备份策略
- 对外部调用失败场景启用熔断与降级
